# 写作规范 -- 防乱码规则

> 加载时机: 创建/编辑任何 workspace 文件时

## 根因

NAS 文件浏览器（WebDAV/SMB）对无 BOM 文件默认用 GBK 解析，导致中文 100% 乱码。

## 强制规则

1. **所有 .md 文件必须加 UTF-8 BOM**（`\xEF\xBB\xBF`）作为文件开头
2. **文件编码**: UTF-8 with BOM，换行符 LF
3. **标题行分隔符**: 使用 `--`（两个 ASCII 横杠），禁止 EM DASH (U+2014) / EN DASH (U+2013) / 全角横杠 (U+FF0D)
4. **标题行禁止 Emoji**（4 字节 UTF-8 字符在 NAS 文件系统可能导致问题）
5. **正文**: 中文/全角符号/Emoji 均可自由使用
6. **禁止编码**: UTF-16 / GBK / 其他非 UTF-8 编码
7. **目录树符号**: 使用 `+--` 和 `|` 代替 Unicode 制表符（`├── └── │`），避免 NAS 渲染异常
8. **SQL/代码文件不加 BOM**（MySQL 客户端 / 编译器兼容性），BOM 仅针对 .md 文件

## 历史踩坑

- 无 BOM 文件在 NAS 文件浏览器中 100% 乱码，已通过全量重写 BOM 修复
- SQL 文件加 BOM 导致 MySQL 客户端报语法错误，因此 BOM 只针对 .md
