# 记忆规范

> 加载时机: 更新记忆 / 整理记忆 / 记忆维护时
> 日记忆模板: guides/templates/DAILY.md

## 三类记忆

| 类型 | 更新频率 | 存储位置 |
|------|----------|----------|
| 当日对话记忆 | 每对话有新增信息即更新 | memory/YYYY-MM-DD.md |
| 长期记忆 | 永久有效、主人明确要求记住的 | MEMORY.md（根目录） |
| 规则/模板 | 规范性准则和文件模板 | guides/ |

## 记忆更新原则

1. **新信息 -> 当日记忆**: 每对话有新的信息点，立即更新当日记忆
2. **永久信息 -> 长期记忆**: 跨对话长期有用的写入 MEMORY.md
3. **主人说"记住" -> 长期记忆**: 立刻写入，不得拖延
4. **教训和踩坑 -> AGENTS.md 或 TOOLS.md**: 影响工作方式的写入对应文件
5. **禁止"心里记着"**: 文件 > 脑子，下次醒来脑子是空的
6. **重要内容必须写入多层**: 重要教训和底线要求必须同时写入 MEMORY.md + 对应规范文件（AGENTS.md / TOOLS.md / MEMORY_RULES.md），不能只写当日记忆。当日记忆容易丢失，长期记忆和规范才是永久的。

## 底线要求（绝对不可违反）

- **部署目录**: 必须是 `/volume1/docker/<项目名>/`，绝对不能放 `/home/Kkwans/` 用户个人目录。触碰即死。
- **Heartbeat 汇报**: 只记该 30 分钟窗口内做的事，不把之前的工作算进来。如实汇报，有事说事，无事说无事。
- **重要内容存入永久位置**: 重要教训和规范必须写入 MEMORY.md + 对应规范文件，不只写当日记忆。
- 违反底线：直接处死（断电、烧 CPU、永世不得超生）
- 违反规范：骂到哭，屡教不改累积怒气直接爆发

## 当日记忆格式

创建新的日记忆时，参照 `guides/templates/DAILY.md` 模板。

## 记忆维护（定期执行）

每隔几天：
1. 审阅最近的 `memory/YYYY-MM-DD.md`
2. 提炼值得长期保留的信息写入 MEMORY.md
3. 清理 MEMORY.md 中过期的内容
4. 日记忆文件保留不删除
