# 2026-05-03 日记忆

## film-forest 项目关键进展

### 1. MySQL Docker 自动重启配置 ✅
- 执行 `docker update --restart always mysql8` 修复了 NAS 重启后 MySQL 不自启的问题
- 验证：`docker inspect mysql8` → RestartPolicy `{always 0}`
- 影响：所有项目（film-forest、interview-master）共用 mysql8 容器，重启后自动恢复

### 2. 爬虫 CloudFlare 绕过问题（进行中）
- **症状**：列表页（`/vs/`、`/zongyi/`、`/dongman/`）返回 138 字节 CloudFlare JS 挑战页，导致 `total=0`
- **详情页**（`/mv/`）：正常工作，能抓取完整 HTML（1-8KB）
- **原因**：CloudFlare 的 JS 挑战在浏览器端执行，HTTP 客户端（Jsoup）无法绕过
- **状态**：待解决，可能需要 Selenium/Puppeteer 或换数据源

### 3. `extractTextByLabel` 解析 bug 修复 ✅（重要！）
- **问题**：pkmp4.xyz HTML 结构中，导演/主演 label 在 `<span class="text-overflow">` 内，但名字在同级的 `<a>` 标签内（不在同 span 内）
- **旧代码**：`parent.hasClass("text-overflow")` 判断失败，导致演员/导演字段为空
- **修复**：遍历 parentDiv 的所有子元素，按顺序收集 label 后的 `<a>` 标签文本
- **验证**：movie 490613 → director: ["菲尔·罗德", "克里斯托弗·米勒"]，actor: ["瑞恩·高斯林", "桑德拉·惠勒"...] ✅
- **推送**：`bddf9f6b fix(crawler): extractTextByLabel 修复导演/主演解析`

### 4. 仍未修复的问题
- **genre 提取**：剧情片详情页 genre 字段为空（标签结构与电影不同）
- **region 提取**：所有类型都失败！原因：`a[href^='/ms/1--']` 只匹配 3+ dash 的 genre href，但 region href 是 `/ms/1-美国----------.html`（单 dash），匹配失败
- **episodes 表**：0 条记录，drama 详情页的剧集信息还没对接

### 5. 调度器异步线程修复 ✅
- **问题**：`@Scheduled` 单线程被慢 HTTP 请求阻塞，新爬虫任务无法触发
- **修复**：`new Thread(() -> { scheduleService.startCrawler(id); }).start()` 异步执行
- **推送**：`7d0ee58 fix: async thread per crawl task`

### 6. GitHub 推送状态
- **本地已保存的提交**（未 push）：
  - `643f731`：refactor(crawler): 降低 HTTP-FETCH 日志级别
  - `a6e2aeb`：docs: update film-forest progress
  - `bddf9f6b`：fix(crawler): extractTextByLabel 修复导演/主演解析
- **推送失败原因**：NAS 环境 git push 超时（SIGTERM），可能是网络问题
- **影响**：这些提交在本地，未丢失，需要网络恢复后手动 push
