### 教训
- NAS 文件由 Docker root 拥有，需要 sudo 操作
- scp 不支持直接写入 NAS 目录，用 ssh+cat 管道上传
- sandbox 环境 git push 超时是已知问题
- **构建顺序红线**：必须先修改代码再构建，不能反过来
- **容器挂载路径**：Film-Forest 大写 D 和 F，docker 小写
- **部署前自测**：构建后必须验证新代码在产物中
