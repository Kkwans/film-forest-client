# 2026-05-04 日记忆

## 影视森林前端 API 代理修复（P0-4）✅

### 问题根因
- client-ui 和 admin-ui 的 API 地址硬编码了 Tailscale IP `100.106.29.60:8080/8081`
- 用户手机不在 Tailscale 网络时，浏览器端 API 请求全部失败
- 症状：用户端电影列表无数据、详情页"电影不存在"、管理端仪表盘"加载中"

### 修复方案
1. `next.config.ts` 添加 rewrites 代理 `/api/*` → 后端
2. `api.ts` 改为相对路径，通过 Next.js server 转发
3. Dockerfile 升级 `node:20-alpine`（Next.js 16 要求 Node >=20.9.0）
4. client-ui 使用 standalone 模式部署（`node server.js`）
5. admin-ui 使用 `npx next start` 部署

### client-ui Dockerfile（standalone 模式）
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY .next/standalone ./
COPY .next/static ./.next/static/
COPY public ./.next/public/
EXPOSE 3000
CMD ["node", "server.js"]
```

### admin-ui Dockerfile（next start 模式）
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --omit=dev
COPY .next ./
COPY next.config.ts ./
EXPOSE 3001
CMD ["npx", "next", "start", "-p", "3001"]
```

### 验证
- `/api/movies` 通过 3000 端口返回数据 ✅
- `/api/content/stats` 通过 3001 端口返回数据 ✅
- `/api/crawler/status` 通过 3001 端口返回数据 ✅

### 注意事项
- 从 OpenClaw 容器内部无法 SSH 到 NAS（exec 环境无 sshpass）
- 需要从 NAS 上直接操作 Docker 命令
- container 文件系统只读，需要重建镜像才能更新
