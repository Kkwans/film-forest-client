# 2026-05-08 日记忆

## UGOS Docker 面板注册机制调研

### 问题
CLI 创建的 docker compose 项目（MySQL、Film-Forest）不在 UGOS Docker 面板中显示，而主人自己通过面板 UI 创建的 Tailscale 可以显示。

### 核心发现
- 面板通过 SQLite 数据库管理 compose 项目，不是扫描目录
- **数据库路径**: `/volume1/@appstore/com.ugreen.docker/db/docker_info_log.db`
- **表名**: `compose`
- 面板 UI 创建项目 → 自动注册到数据库
- CLI 创建项目 → 不会自动注册，需要手动插入

### 解决方案
手动往 compose 表插入记录即可让面板识别 CLI 创建的项目：
```bash
sudo sqlite3 /volume1/@appstore/com.ugreen.docker/db/docker_info_log.db \
  "INSERT INTO compose (created_at, updated_at, name, state, path, content, app_id, container_num) 
   VALUES (datetime('now'), datetime('now'), '<项目名>', 1, '<compose路径>', '', '', <容器数>);"
```

### 已完成
- MySQL 和 Film-Forest 已注册到面板，正常显示
- DOCKER_RULES.md 4.6 节已更新为面板注册规范
