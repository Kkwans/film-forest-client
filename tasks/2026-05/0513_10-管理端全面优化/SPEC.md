- 乱码数据修复: UPDATE crawler_schedule SET name = CONVERT(CAST(name AS BINARY) USING utf8mb4)

### 2.2 表单弹窗重构（T2）

**问题**:
- 弹窗样式丑，排版差
- 输入框超出弹窗边界（特别是移动端）
- 表单字段不完整（内容管理新增/编辑缺失很多字段）

**方案**:
- 重新设计 Modal 组件，支持响应式
- 内容管理表单扩展：增加 language/duration/releaseDate/alias/scoreImdb/scoreRt 等字段
- 表单布局: 标签在上方，输入框 100% 宽度，移动端单列

### 2.3 组件库升级（T3）

**问题**: 浏览器原生 select/alert/confirm 很丑

**方案**: 参照用户端前端设计
- 自定义 Select 组件（下拉列表）
- 已有 Toast 组件 → 优化样式
- 已有 Dialog 组件 → 优化样式
- 所有浏览器原生组件替换为自定义

### 2.4 内容管理优化（T4）

**问题**:
- 列表排版需要优化
- 缺少"查看详情"按钮
- 新增/编辑表单字段不完整
- 操作后不刷新

**方案**:
- 列表: 增加详情按钮，优化状态按钮样式
- 详情 Modal: 展示所有字段（参照电影详情页设计）
- 表单: 扩展字段（language/duration/releaseDate/alias/scoreImdb/scoreRt/writer）
- 操作后: 自动调用 fetchItems() 刷新

### 2.5 爬虫配置增强（T5）

**问题**:
- genre 手动输入逗号分隔 → 应从 DB 查询多选
- Cron 选项太少 → 需要高度自定义
- 资源来源未关联爬虫

**方案**:
- genre: 已有 getGenres API，前端用 checkbox 多选 ✅（已实现）
- Cron: 增加更多预设 + 自定义输入 + Cron 表达式预览
- 资源来源: crawler_schedule 增加 source_id 字段关联 resource_source 表

### 2.6 爬虫保存 JSON 报错修复（T6）

**问题**: `genre_filter` 列期望 JSON 格式，但传入了空字符串或非法格式

**方案**:
- 后端: CrawlerScheduleServiceImpl.saveSchedule() 增加 normalizeGenreFilter()
  - 空字符串 → null
  - 逗号分隔 → JSON 数组
  - 已是 JSON → 直接使用
- 前端: 提交时确保 genreFilter 是 JSON 数组格式

### 2.7 数据统计页面优化（T7）

**问题**: 与仪表盘区分度不够，布局不够好

**方案**:
- 重新设计页面布局
- 增加更多统计维度
- 视觉上与仪表盘明确区分

### 2.8 数据统计图表增强（T8）

**问题**:
- 百分比 bug（只有综艺显示正确百分比）
- 图表类型不够丰富

**方案**:
- 修复百分比计算逻辑
- 增加图表类型:
  - 饼图: 内容类型分布
  - 柱状图: 每日/每周/每月爬取量
  - 折线图: 数量增长趋势
- 使用 recharts 库（已在项目中）

### 2.9 资源管理完善（T9）

**问题**:
- 磁力/网盘/来源列表为空
- 缺少默认来源配置
- 来源未关联爬虫

**方案**:
- 后端: 确保 API 返回正确数据
- 前端: 资源列表页面完善
- 默认数据: resource_source 表预置七味网等来源
- 关联: crawler_schedule.source_id → resource_source.id

### 2.10 系统设置页面优化（T10）

**问题**: 布局样式不好看

**方案**: 重新设计卡片式布局，分组展示设置项

### 2.11 爬虫脚本优化（T11）— 核心任务

**问题**:
- 需要基于七味网调研文档完善爬虫
- 断点续爬支持
- 数据去重逻辑
- 健壮性要求

**方案**:
- 参照 EXPLORE.md 的选择器和 URL 规则
- CrawlerCore 增加:
  - last_crawled_page/last_crawled_id 断点字段（已有）
  - 增量爬取: 通过 contentId 去重
  - 异常处理: 重试机制 + 错误日志
  - 5 种内容类型全覆盖: movie/drama/variety/anime/short_drama

### 2.12 登录校验 + 通知告警（T12）

**方案**:
- JWT 认证（已有实现）
- 前端: AuthProvider + 路由守卫（已有）
- 通知: 站内通知表 + 通知组件
- 扩展预留: 邮件通知接口
