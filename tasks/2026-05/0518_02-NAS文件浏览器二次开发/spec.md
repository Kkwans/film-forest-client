# SPEC -- NAS 文件浏览器二次开发

## 1. 目标与价值

- **解决什么问题**: FileBrowser 功能完善但 UI/MDEditing 体验差，需要定制化
- **为什么要做**: NAS 文件管理是日常高频操作，好的体验直接影响效率
- **预期收益**: 获得一个功能完整、UI 精美、Markdown 编辑体验优秀的 NAS 文件管理器

## 2. 上下文与约束

- **基础项目**: filebrowser/filebrowser (GitHub, Apache 2.0)
- **前端**: Vue 3 + TypeScript + Vite
- **后端**: Go
- **当前版本**: v2.31.2
- **部署**: Docker on NAS (ARM64), 端口 8888
- **代码托管**: GitHub Kkwans/nas-file-browser
- **约束**: 保持与上游的兼容性，改动尽量模块化

## 3. 功能需求

### F1: Markdown 渲染美化 (P0)
- **描述**: 替换默认 5 行 CSS，提供专业级 Markdown 渲染样式
- **渲染元素**: 标题(h1-h6)、段落、代码块(暗色主题)、行内代码、表格、引用块、列表、图片、链接、分割线、复选框
- **主题**: 亮色 + 暗色适配
- **验收**: 打开任意 .md 文件，渲染效果美观专业

### F2: Markdown 编辑器升级 (P0)
- **描述**: 用 Vditor 替换 Ace + marked 简单预览
- **模式**:
  - IR（实时渲染）: Typora 风格，默认模式
  - SV（源码+预览分屏）: 左源码右预览
  - 阅读模式: 纯阅读，无编辑干扰
  - 源码模式: 纯源码编辑
- **功能**: 自动保存、Ctrl+S 保存、字数统计、大纲导航
- **验收**: 打开 .md 文件默认进入 IR 模式，可切换

### F3: UI 全局样式优化 (P1)
- **描述**: 优化整体视觉效果
- **范围**: 文件列表、侧边栏、导航栏、设置页、登录页
- **优化点**: 间距、圆角、阴影、颜色、字体、动画
- **验收**: 整体视觉现代精致

### F4: 中文本地化完善 (P1)
- **描述**: 完善中文翻译，确保所有界面元素中文化
- **范围**: 按钮、提示、错误信息、设置项
- **验收**: 切换到中文后无遗漏

### F5: 文件预览增强 (P2)
- **描述**: 增强各类文件的预览体验
- **范围**: 代码文件语法高亮、PDF 预览、Office 文档预览
- **验收**: 常见文件类型有良好预览

### F6: 批量操作优化 (P2)
- **描述**: 优化批量选择、移动、复制、删除体验
- **验收**: 批量操作流畅直观

## 4. 非功能需求

- **性能**: 文件列表加载 < 500ms，编辑器启动 < 2s
- **兼容性**: Chrome/Safari/Firefox，移动端响应式
- **构建**: Docker 镜像 < 100MB，构建时间 < 10min
- **可维护性**: 改动模块化，方便合并上游更新

## 5. 验收标准

- [ ] Markdown 渲染美观（代码块、表格、引用等样式专业）
- [ ] Vditor 集成完成（IR/分屏/源码/阅读四种模式可用）
- [ ] 暗色模式下 Markdown 渲染正常
- [ ] 中文界面无遗漏
- [ ] Docker 镜像可正常构建和部署
- [ ] 代码推送到 GitHub nas-file-browser 仓库
- [ ] NAS 上 8888 端口运行自定义版本
