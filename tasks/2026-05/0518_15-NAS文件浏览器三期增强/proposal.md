# PROPOSAL -- NAS 文件浏览器三期增强

## 背景

NAS 文件浏览器（nas-file-browser）已完成二期开发（Markdown 渲染、Vditor 编辑器、UI 样式优化、中文本地化），但存在以下核心问题：

1. **存储卷展示不完整** — 当前仅挂载 `/volume1/@home/Kkwans`，无法浏览 volume2、外置存储、网络存储等
2. **密码策略过于严格** — 内置密码强度检查拒绝简单密码，用户体验差
3. **错误提示为英文** — 后端返回的错误信息未汉化
4. **缺乏目录分类** — 未区分个人文件夹、共享文件夹、系统文件夹
5. **缺乏风险等级标识** — 高危/中危/低危文件夹未区分

## 目标

参考绿联 NAS 官方文件管理软件（UGOS File Manager）的设计理念，实现：

1. **全存储卷展示** — admin 用户可浏览所有存储卷（volume1、volume2、外置 USB、网络存储）
2. **目录分类体系** — 个人文件夹 vs 共享文件夹，清晰的目录结构
3. **风险等级标识** — 高危（系统配置）、中危（需了解才能修改）、低危（用户文件）
4. **密码策略优化** — 允许简单密码（6 位），移除内置强度检查
5. **错误信息汉化** — 所有用户可见的错误信息使用中文
6. **核心功能优先** — 文件浏览/预览/编辑性能 > UI 样式

## 范围

### In Scope
- 存储卷自动发现与挂载
- 目录分类与风险等级系统
- 密码策略源码修改
- 错误信息汉化
- 目录收藏功能（Phase 2）
- 目录标签功能（Phase 2）

### Out of Scope
- 文件搜索功能增强（已有基础实现）
- 用户权限精细化（保持 admin 全权限）
- 移动端专属 UI（保持响应式）

## 技术约束

- 前端：Vue 3 + Vite + TypeScript
- 后端：Go（filebrowser 原版）
- 部署：Docker 容器，ARM64 架构
- 存储：绿联 DH4300Plus NAS，多存储卷

## 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Go 源码修改复杂 | 高 | 先调研 filebrowser 源码结构，定位关键文件 |
| 多存储卷挂载性能 | 中 | 使用符号链接或 UnionFS，避免多次挂载 |
| 密码策略绕过安全风险 | 低 | 仅限 admin 用户，内网使用 |

## 预计工作量

- Phase 1（核心修复）：2-3 小时
- Phase 2（目录分类 + 风险等级）：4-6 小时
- Phase 3（收藏 + 标签）：3-4 小时

## 依赖

- NAS 文件系统结构分析（volume1、volume2、外置存储路径）
- 绿联 NAS 官方文件管理软件设计参考
- filebrowser Go 源码分析
