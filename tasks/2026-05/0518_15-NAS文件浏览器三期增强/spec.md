# SPEC -- NAS 文件浏览器三期增强

## 1. 目标与价值

### 核心问题
当前 NAS 文件浏览器仅能浏览单个用户目录，无法满足管理员对全存储系统的管理需求。密码策略过于严格，错误信息为英文，缺乏目录分类和风险标识。

### 预期价值
- 管理员可一站式浏览和管理所有存储资源
- 目录分类和风险等级降低误操作风险
- 简化密码策略提升用户体验
- 中文错误信息降低使用门槛

## 2. 上下文与约束

### 技术栈
- 前端：Vue 3.5 + Vite 8 + TypeScript 5.9
- 后端：Go（filebrowser 原版，需 fork 修改）
- 部署：Docker 容器，ARM64 架构（绿联 DH4300Plus）
- 存储：多存储卷（volume1、volume2、外置 USB、网络存储）

### 现有架构
```
nas-file-browser/
├── frontend/          # Vue 3 前端
├── docker/            # Docker 配置
├── Dockerfile.custom  # 两阶段构建（前端 + Go 后端）
└── docker-compose.custom.yml
```

### 约束
- Go 后端需要 fork 原版 filebrowser 并修改
- 前端需要扩展目录树组件支持多存储卷
- 密码策略修改需要同时修改后端验证逻辑和前端提示

## 3. 功能需求

### FR-1: 全存储卷展示

**描述：** admin 用户可浏览 NAS 上所有存储卷，包括系统存储和外置存储。

**存储卷类型：**
| 类型 | 路径 | 说明 |
|------|------|------|
| 系统存储 volume1 | /volume1 | 主存储卷 |
| 系统存储 volume2 | /volume2 | 扩展存储卷（如有） |
| 外置 USB | /volumeUSB1, /volumeUSB2, ... | USB 外置存储 |
| 网络存储 | /volumeSATA, /volumeNVMe, ... | 网络挂载存储 |
| Docker 存储 | /volume1/@docker | Docker 数据 |

**实现方式：**
- 后端：修改 Go 代码，支持多 root 目录
- 前端：扩展侧边栏，显示存储卷列表
- 挂载：docker-compose 挂载 `/volume1:/volume1`, `/volume2:/volume2` 等

**验收标准：**
- [ ] admin 用户登录后可看到所有存储卷
- [ ] 每个存储卷显示总容量和已用空间
- [ ] 可在存储卷之间自由切换浏览

### FR-2: 目录分类体系

**描述：** 参考绿联 UGOS 文件管理器，将目录分为个人文件夹和共享文件夹。

**分类结构：**
```
├── 个人文件夹（Personal）
│   ├── HOME/          # 用户主目录
│   ├── Desktop/       # 桌面文件
│   ├── Documents/     # 文档
│   ├── Downloads/     # 下载
│   ├── Music/         # 音乐
│   ├── Photos/        # 照片
│   └── Videos/        # 视频
├── 共享文件夹（Shared）
│   ├── 公共文件夹/     # 所有用户可访问
│   ├── 团队文件夹/     # 特定用户组可访问
│   └── 临时共享/       # 临时共享文件
└── 系统文件夹（System）
    ├── @appstore/     # 应用数据
    ├── @docker/       # Docker 数据
    ├── @home/         # 用户主目录根
    └── @tmp/          # 临时文件
```

**实现方式：**
- 后端：新增目录分类配置文件 `categories.json`
- 前端：侧边栏分组显示，不同分类用不同图标和颜色
- 配置：支持自定义分类规则

**验收标准：**
- [ ] 侧边栏显示个人文件夹、共享文件夹、系统文件夹三个分组
- [ ] 每个分组有独立的图标和颜色标识
- [ ] 支持自定义分类规则

### FR-3: 风险等级标识

**描述：** 根据目录的重要性和修改风险，分为高危、中危、低危三个等级。

**风险等级定义：**
| 等级 | 颜色 | 目录示例 | 操作限制 |
|------|------|---------|---------|
| 高危 | 🔴 红色 | /volume1/@docker, /volume1/@appstore, /etc | 只读，修改需二次确认 |
| 中危 | 🟡 黄色 | /volume1/Docker, /volume1/@home | 可修改，但需确认 |
| 低危 | 🟢 绿色 | /volume1/@home/Kkwans, /volume1/Download | 自由操作 |

**实现方式：**
- 后端：新增风险等级配置文件 `risk-levels.json`
- 前端：目录列表显示风险等级图标，高危目录操作弹出确认对话框
- 配置：支持自定义风险等级规则

**验收标准：**
- [ ] 高危目录显示红色警告图标
- [ ] 高危目录的删除/修改操作需要二次确认
- [ ] 中危目录显示黄色提示图标
- [ ] 低危目录正常显示

### FR-4: 密码策略优化

**描述：** 允许用户设置简单密码（最低 6 位），移除内置的密码强度检查。

**当前问题：**
- 后端 Go 代码有内置的密码强度检查（`password is too easy`）
- 前端显示英文错误信息
- 用户无法设置简单密码

**修改方案：**
1. Fork filebrowser 源码
2. 定位密码验证逻辑（`users.go` 或 `auth.go`）
3. 移除或注释掉密码强度检查代码
4. 保留最小长度检查（6 位）
5. 前端错误信息汉化

**验收标准：**
- [ ] 用户可设置 6 位简单密码（如 `123456`）
- [ ] 密码修改失败时显示中文错误信息
- [ ] 最小长度限制为 6 位

### FR-5: 错误信息汉化

**描述：** 所有用户可见的错误信息使用中文显示。

**需要汉化的错误信息：**
- `password is too short, minimum length is {n}` → `密码太短，最少需要 {n} 位`
- `password is too easy` → `密码强度不够，请使用更复杂的密码`
- `401 Unauthorized` → `未授权，请重新登录`
- `403 Forbidden` → `没有权限执行此操作`
- `404 Not Found` → `请求的资源不存在`
- `500 Internal Server Error` → `服务器内部错误`

**实现方式：**
- 后端：修改 Go 代码，返回中文错误信息
- 前端：i18n 国际化配置

**验收标准：**
- [ ] 所有用户可见的错误信息为中文
- [ ] 错误信息清晰易懂

### FR-6: 目录收藏功能（Phase 2）

**描述：** 用户可收藏常用目录，快速访问。

**功能点：**
- 收藏/取消收藏目录
- 侧边栏显示收藏列表
- 收藏目录拖拽排序
- 收藏数据持久化（localStorage）

**验收标准：**
- [ ] 可收藏/取消收藏目录
- [ ] 侧边栏显示收藏列表
- [ ] 收藏数据在刷新后保持

### FR-7: 目录标签功能（Phase 2）

**描述：** 用户可给目录打标签，用于分类和快速筛选。

**功能点：**
- 创建/编辑/删除标签
- 给目录分配标签
- 按标签筛选目录
- 标签颜色自定义

**验收标准：**
- [ ] 可创建/编辑/删除标签
- [ ] 可给目录分配标签
- [ ] 可按标签筛选目录

## 4. 非功能需求

### NFR-1: 性能
- 目录列表加载时间 < 500ms
- 文件预览加载时间 < 1s
- 大目录（>1000 文件）分页加载

### NFR-2: 安全
- 高危目录操作需要二次确认
- 密码存储使用 bcrypt 加密
- 会话超时时间 2 小时

### NFR-3: 可用性
- 中文界面，所有提示信息为中文
- 响应式设计，支持移动端
- 暗色模式支持

### NFR-4: 可维护性
- 代码结构清晰，模块化设计
- 配置文件外置，支持自定义
- 日志记录完整，便于排查问题
